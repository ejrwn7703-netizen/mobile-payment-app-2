## 1. 서론

### 1.1 목적

본 문서는 편의점 등 소매점에서 사용되는 무인계산 모바일 애플리케이션의 소프트웨어 요구사항을 정의한다. 이 문서는 개발팀, 테스트팀, 프로젝트 관리자, 그리고 이해관계자들이 시스템의 기능과 제약사항을 명확히 이해할 수 있도록 작성되었다.

### 1.2 범위

**제품명**: SelfPay (무인계산 모바일 앱)

**주요 기능**:
- 모바일 기기를 통한 상품 바코드 스캔
- 실시간 상품 정보 조회 및 장바구니 관리
- 간편결제(네이버페이, 카카오페이) 처리
- 부정사용 방지를 위한 다층 보안 시스템
- 매장 관리자용 실시간 모니터링 시스템

**적용 범위**: 편의점, 소규모 슈퍼마켓, 무인매장

## 2. 전체 설명

### 2.1 제품 관점

SelfPay는 기존 매장의 POS 시스템과 연동되는 **보완적 솔루션**이다. 시스템 구성은 다음과 같다:

```
[모바일 앱] ↔ [백엔드 API 서버] ↔ [매장 POS 시스템]
     ↓              ↓                    ↓
[사용자]      [클라우드 DB]         [매장 관리자]
     ↓              ↓                    ↓
[결제 시스템] ↔ [보안 시스템] ↔ [모니터링 시스템]
```

### 2.2 제품 기능

**주요 기능 개요**:

1. **F1. 상품 스캔 및 인식**
   - 바코드/QR코드 스캔
   - 상품 정보 실시간 조회
   - 다중 상품 스캔 지원

2. **F2. 장바구니 관리**
   - 상품 추가/삭제/수량 변경
   - 실시간 가격 계산
   - 할인 적용

3. **F3. 결제 처리**
   - 간편결제 연동
   - 결제 검증 및 승인
   - 전자영수증 발급

4. **F4. 보안 및 부정방지**
   - 중량 검증 시스템
   - AI 기반 행동 분석
   - 출구 인증 시스템

5. **F5. 사용자 관리**
   - 회원가입/로그인
   - 구매 이력 관리
   - 개인정보 보호

6. **F6. 매장 관리**
   - 실시간 매출 모니터링
   - 재고 관리
   - 보안 이벤트 관리

### 2.3 사용자 특성

| 사용자 유형 | 특성 | 기술 수준 | 사용 빈도 |
|-------------|------|-----------|-----------|
| **일반 고객** | 20-60세, 스마트폰 사용자 | 초급-중급 | 주 2-3회 |
| **매장 직원** | 20-50세, 매장 운영 경험 | 중급 | 매일 |
| **매장 관리자** | 30-60세, 매장 관리 경험 | 중급-고급 | 매일 |
| **시스템 관리자** | IT 전문가 | 고급 | 필요시 |

### 2.4 제약사항

**REQ-CON-001**: **플랫폼 제약사항**
- Android 8.0 (API 26) 이상 지원
- iOS 12.0 이상 지원
- 카메라 기능 필수

**REQ-CON-002**: **네트워크 제약사항**
- 인터넷 연결 필수 (Wi-Fi 또는 모바일 데이터)
- 최소 대역폭: 1Mbps

**REQ-CON-003**: **법적 제약사항**
- 개인정보보호법 준수
- 전자상거래법 준수
- PCI DSS 표준 준수

**REQ-CON-004**: **비즈니스 제약사항**
- 매장 운영시간 내에만 서비스 이용 가능
- 성인 인증이 필요한 상품 제외

### 2.5 가정과 종속성

**가정사항**:
- 사용자는 스마트폰 기본 조작이 가능하다
- 매장에는 안정적인 Wi-Fi 환경이 제공된다
- 매장 직원이 상황 발생시 대응 가능하다

**종속성**:
- 네이버페이/카카오페이 API 가용성
- 매장 POS 시스템과의 호환성
- 클라우드 서비스 가용성

## 3. 시스템 요구사항

### 3.1 기능 요구사항

#### 3.1.1 상품 스캔 및 인식 (F1)

**REQ-FUNC-001**: **바코드 스캔 기능**
- **설명**: 시스템은 모바일 기기의 카메라를 통해 상품 바코드를 스캔할 수 있어야 한다
- **입력**: 카메라로 촬영된 바코드 이미지
- **처리**: 바코드 디코딩 및 상품 정보 조회
- **출력**: 상품 정보 (상품명, 가격, 이미지)
- **우선순위**: 필수 (Must Have)

**REQ-FUNC-002**: **바코드 인식 정확도**
- **설명**: 바코드 인식 정확도는 99.5% 이상이어야 한다
- **측정방법**: 1000회 스캔 테스트에서 995회 이상 정확 인식
- **우선순위**: 필수 (Must Have)

**REQ-FUNC-003**: **스캔 응답시간**
- **설명**: 바코드 스캔 후 상품 정보 표시까지 1초 이내여야 한다
- **측정방법**: 95% 요청이 1초 이내 응답
- **우선순위**: 필수 (Must Have)

**REQ-FUNC-004**: **지원 바코드 형식**
- **설명**: EAN-13, EAN-8, Code-128, QR코드 형식을 지원해야 한다
- **우선순위**: 필수 (Must Have)

**REQ-FUNC-005**: **상품 정보 조회 실패 처리**
- **설명**: 상품 정보 조회 실패시 적절한 오류 메시지를 표시해야 한다
- **오류 유형**: 네트워크 오류, 상품 미등록, 서버 오류
- **우선순위**: 필수 (Must Have)

#### 3.1.2 장바구니 관리 (F2)

**REQ-FUNC-006**: **상품 추가 기능**
- **설명**: 스캔된 상품을 장바구니에 추가할 수 있어야 한다
- **제약사항**: 동일 상품 최대 99개까지 추가 가능
- **우선순위**: 필수 (Must Have)

**REQ-FUNC-007**: **수량 변경 기능**
- **설명**: 장바구니의 상품 수량을 변경할 수 있어야 한다
- **입력방법**: +/- 버튼, 직접 입력
- **우선순위**: 필수 (Must Have)

**REQ-FUNC-008**: **상품 삭제 기능**
- **설명**: 장바구니에서 상품을 삭제할 수 있어야 한다
- **확인절차**: 삭제 확인 다이얼로그 표시
- **우선순위**: 필수 (Must Have)

**REQ-FUNC-009**: **실시간 가격 계산**
- **설명**: 장바구니 내용 변경시 총 금액을 실시간으로 계산해야 한다
- **포함사항**: 상품가격, 할인, 부가세
- **우선순위**: 필수 (Must Have)

**REQ-FUNC-010**: **장바구니 임시 저장**
- **설명**: 앱 종료 후 재시작시 장바구니 내용이 유지되어야 한다
- **저장기간**: 24시간
- **우선순위**: 높음 (Should Have)

#### 3.1.3 결제 처리 (F3)

**REQ-FUNC-011**: **결제 수단 선택**
- **설명**: 네이버페이, 카카오페이 중 선택하여 결제할 수 있어야 한다
- **우선순위**: 필수 (Must Have)

**REQ-FUNC-012**: **결제 요청 생성**
- **설명**: 선택한 결제 수단으로 결제 요청을 생성해야 한다
- **포함정보**: 상품목록, 총금액, 매장정보, 사용자정보
- **우선순위**: 필수 (Must Have)

**REQ-FUNC-013**: **결제 승인 처리**
- **설명**: PG사로부터 결제 승인 결과를 처리해야 한다
- **처리시간**: 5초 이내
- **우선순위**: 필수 (Must Have)

**REQ-FUNC-014**: **전자영수증 발급**
- **설명**: 결제 완료시 전자영수증을 생성하고 표시해야 한다
- **포함정보**: 법정 필수 항목 + QR코드
- **형식**: PDF 및 이미지
- **우선순위**: 필수 (Must Have)

**REQ-FUNC-015**: **결제 실패 처리**
- **설명**: 결제 실패시 원인별 적절한 안내 메시지를 표시해야 한다
- **재시도**: 최대 3회 재시도 허용
- **우선순위**: 필수 (Must Have)

#### 3.1.4 보안 및 부정방지 (F4)

**REQ-FUNC-016**: **중량 검증**
- **설명**: 스캔된 상품의 총 중량과 실제 측정 중량을 비교 검증해야 한다
- **허용오차**: ±5% 또는 50g 중 큰 값
- **우선순위**: 필수 (Must Have)

**REQ-FUNC-017**: **출구 인증**
- **설명**: 결제 완료 후 출구에서 QR코드를 통해 인증해야 한다
- **인증방법**: 앱 생성 QR코드 스캔
- **유효시간**: 10분
- **우선순위**: 필수 (Must Have)

**REQ-FUNC-018**: **의심행동 탐지**
- **설명**: AI 시스템을 통해 의심스러운 행동을 탐지해야 한다
- **탐지 대상**: 스캔 없는 상품 은닉, 바코드 조작 등
- **정확도**: 95% 이상
- **우선순위**: 높음 (Should Have)

**REQ-FUNC-019**: **보안 이벤트 알림**
- **설명**: 보안 이벤트 발생시 매장 관리자에게 실시간 알림을 전송해야 한다
- **알림 방법**: 푸시 알림, SMS, 이메일
- **응답시간**: 5초 이내
- **우선순위**: 필수 (Must Have)

#### 3.1.5 사용자 관리 (F5)

**REQ-FUNC-020**: **사용자 인증**
- **설명**: 전화번호 기반 SMS 인증으로 회원가입/로그인해야 한다
- **인증코드**: 6자리 숫자, 유효시간 3분
- **우선순위**: 필수 (Must Have)

**REQ-FUNC-021**: **소셜 로그인**
- **설명**: 네이버, 카카오, 구글 계정으로 로그인할 수 있어야 한다
- **우선순위**: 높음 (Should Have)

**REQ-FUNC-022**: **구매 이력 조회**
- **설명**: 사용자의 과거 구매 내역을 조회할 수 있어야 한다
- **조회 기간**: 최근 1년
- **정렬**: 최신순
- **우선순위**: 높음 (Should Have)

**REQ-FUNC-023**: **개인정보 관리**
- **설명**: 사용자가 개인정보를 조회/수정할 수 있어야 한다
- **수정 가능 항목**: 닉네임, 알림 설정
- **우선순위**: 높음 (Should Have)

#### 3.1.6 매장 관리 (F6)

**REQ-FUNC-024**: **실시간 매출 모니터링**
- **설명**: 매장 관리자는 실시간 매출 현황을 확인할 수 있어야 한다
- **표시 정보**: 시간대별 매출, 인기 상품, 거래 건수
- **업데이트**: 1분 간격
- **우선순위**: 높음 (Should Have)

**REQ-FUNC-025**: **재고 관리**
- **설명**: 상품별 재고 현황을 관리할 수 있어야 한다
- **기능**: 재고 조회, 수정, 부족 알림
- **우선순위**: 높음 (Should Have)

**REQ-FUNC-026**: **보안 이벤트 관리**
- **설명**: 발생한 보안 이벤트를 조회하고 처리할 수 있어야 한다
- **처리 기능**: 확인, 해결, 메모 추가
- **우선순위**: 필수 (Must Have)

### 3.2 비기능 요구사항

#### 3.2.1 성능 요구사항

**REQ-PERF-001**: **동시 사용자 지원**
- **설명**: 매장당 최소 50명의 동시 사용자를 지원해야 한다
- **측정방법**: 부하 테스트로 검증
- **우선순위**: 필수 (Must Have)

**REQ-PERF-002**: **응답 시간**
- **설명**: API 응답 시간은 95백분위수 기준 2초 이내여야 한다
- **예외**: 결제 처리는 5초 이내
- **우선순위**: 필수 (Must Have)

**REQ-PERF-003**: **시스템 가용성**
- **설명**: 시스템 가용성은 99.5% 이상이어야 한다
- **계산**: 월간 다운타임 3.6시간 이내
- **우선순위**: 필수 (Must Have)

**REQ-PERF-004**: **데이터베이스 성능**
- **설명**: 상품 조회 쿼리는 평균 100ms 이내 응답해야 한다
- **측정조건**: 100만 건 상품 데이터 기준
- **우선순위**: 필수 (Must Have)

**REQ-PERF-005**: **모바일 앱 성능**
- **설명**: 앱 시작 시간은 3초 이내여야 한다
- **측정조건**: 중급 사양 스마트폰 기준
- **우선순위**: 높음 (Should Have)

#### 3.2.2 보안 요구사항

**REQ-SEC-001**: **데이터 전송 암호화**
- **설명**: 모든 데이터 전송은 TLS 1.3 이상으로 암호화되어야 한다
- **적용 범위**: 클라이언트-서버, 서버-PG사 통신
- **우선순위**: 필수 (Must Have)

**REQ-SEC-002**: **개인정보 암호화**
- **설명**: 저장되는 개인정보는 AES-256으로 암호화되어야 한다
- **적용 대상**: 전화번호, 이름, 이메일
- **우선순위**: 필수 (Must Have)

**REQ-SEC-003**: **인증 토큰 관리**
- **설명**: JWT 토큰 기반 인증을 사용해야 한다
- **토큰 만료**: 액세스 토큰 15분, 리프레시 토큰 7일
- **우선순위**: 필수 (Must Have)

**REQ-SEC-004**: **비밀번호 정책**

### 4.2 테스트 전략

#### 4.2.1 단위 테스트
- **범위**: 모든 비즈니스 로직
- **커버리지**: 최소 80%
- **도구**: Jest(JavaScript), JUnit(Java), pytest(Python)

#### 4.2.2 통합 테스트
- **범위**: API 간 연동, 외부 시스템 연동
- **자동화**: CI/CD 파이프라인에 포함
- **도구**: Postman, Newman

#### 4.2.3 성능 테스트
- **부하 테스트**: 동시 사용자 1000명
- **스트레스 테스트**: 시스템 한계 확인
- **도구**: JMeter, LoadRunner

#### 4.2.4 보안 테스트
- **침투 테스트**: 외부 보안 업체 수행
- **취약점 스캔**: OWASP ZAP 활용
- **주기**: 분기별 1회

#### 4.2.5 사용성 테스트
- **대상**: 실제 사용자 20명
- **방법**: 태스크 기반 테스트
- **측정**: 완료율, 소요시간, 만족도

### 4.3 수용 기준

각 요구사항은 다음 기준을 만족해야 인수된다:

1. **기능 요구사항**: 명시된 기능이 정상 동작
2. **성능 요구사항**: 성능 목표치 달성
3. **보안 요구사항**: 보안 테스트 통과
4. **사용성 요구사항**: 사용성 테스트 기준 충족
